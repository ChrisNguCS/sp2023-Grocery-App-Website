Final Deliverables

  1. A mobile or web application with a user-friendly interface and easy-to-use functionality for creating, editing, and deleting grocery items.
  2. A database to store the grocery items, allowing users to access and update their lists from any device.
  3. Login and authentication methods to keep user data and to allow for group collaboration.
  4. A comprehensive documentation package that includes requirements, design, user guides, and release notes.

Risk Assessment

  1. Hardware limitations can cause issues if people are running an older version of iOS or Android.
  2. Debugging React Native can be difficult due to thrrre framework being built using JS, Objective-C, Java, and C/C++.
  3. Testing might be more tedious due to the framework being cross compatible with iOS and Android. This leads to two potential error groups that can lead to more delays down the line.

Version Control

Our plan of version control is GitHub. GitHub will include all code and updates for mobile app.
